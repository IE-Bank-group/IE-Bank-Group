In order to manage our credentails securely, we decided to leverage Azure Key Vault for storing sensitive informations such as usernames, passwords, and deployment tokens. We have separate resource groups for development (dev), user acceptance testing (UAT), and production (prod) enviroments. This segmentation ensures that each environment has its own set of credentails, which adds an extra layer of security and prevents any accidental mixing of secrets between differnt stages of deployment.

For example, in the apayne-kv-dev Key Vault, we securely store secret such as the Container Registry credentials (username and password) and the Static Web App Deployment Token for the frontend. These secrets are critical because they enables our systems to securely connect to the Azure Container Registry and deploy the frontend to the appropriate enviroment. Without these credentials, our deployment process would be interupted, which is why itâ€™s crucial to ensure these secrets are never exposed in code or pushed into GitHub.

The Container Registry credentails (both username and passwords) are needed for accessing and pushing images to our Azure Container Registry. This is particularly importnat as it ensures that only authorized systems or users can upload or pull container images, protecting our infrastrcture from unauthorized access.

The Static Web App Deployment Token is another important credentials we store in the Key Vault. This token allows the frontend application to be deploy to the appropriate Azure static web app. Without this token, deployment would fail, disrupting our continuos integration and deployment pipeline.

As we have divided our resources across dev, UAT, and prod enviroments, storing these secrets separately for each enviroment allows us to manage access with greater precission. For example, different users may have access to the development secrets, but they would not have access to the production credentails. This segmentation helps ensure that production secrets are well guarded and are not accessible by those working on development or testing enviroments.

In summarry, securely managing credentails through Azure Key Vault is an essential practice to ensure the safety of our deployment pipelines. By storing all sensitive information in the Key Vault and managing them per enviroment (dev, UAT, prod), we are enhancing the security of our operations and preventing any accidental leaks or exposures.
