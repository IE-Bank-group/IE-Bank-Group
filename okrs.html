Objectives and Key Results (OKRs)

The following OKRs are designed to guide the development and launch of the MVP for the IE Bank app over the next quarter. Each objective supports critical aspects of the project, from user satisfaction to operational efficiency and security.

Objective 1: Improve User Satisfaction

  • KR 1: Achieve a 90% satisfaction rating based on post-launch user surveys within the first quarter.
  • KR 2: Complete usability testing with a 90% task success rate, ensuring users can use the app easily.
  • KR 3: Increase active user retention by 25% by providing an easy and engaging user experience.
  • KR 4: Resolve 90% of user-reported issues within a week of reporting to maintain user trust and satisfaction.
  • KR 5: Implement a user feedback loop, achieving a 70% response rate to ensure ongoing improvements based on real user insights.

Objective 2: Strengthen Security and Data Protection

  • KR 1: Ensure 100% of user passwords are stored as securely hashed.
  • KR 2: Conduct security testing to identify and resolve at least 95% of critical issues before deployment.
  • KR 3: Achieve a 100% pass rate for all authentication and authorization tests to prevent any unauthorized access.
  • KR 4: Make sure all team member have knowldege of the best security practices.
  • KR 5: Set up automated monitoring for user login attempts to detect and block suspicious activity, with a response time of under 5 minutes.

Objective 3: Enhance Operational Efficiency in Development and Deployment

  • KR 1: Implement a CI/CD pipeline with automated builds and deployments, achieving a 90% reduction in manual deployment tasks.
  • KR 2: Complete each sprint on schedule with at least 95% of planned tasks marked as “done.”
  • KR 3: Automate 80% of test cases for continuous integration, minimizing testing time while ensuring quality.
  • KR 4: Reduce code review and approval time by 50% through streamlined GitHub workflows and team collaboration.
  • KR 5: Maintain an average issue resolution time of under 48 hours to keep development on track.

Objective 4: Increase Product Stability and Reliability

  • KR 1: Achieve 99.9% uptime in the development and UAT environments, with production aiming for the same reliability standard post-launch.
  • KR 2: Conduct load testing to confirm that the system can handle at least 1,000 concurrent users without performance degradation.
  • KR 3: Resolve 95% of critical bugs reported during testing within 72 hours.
  • KR 4: Achieve an average response time of under 2 seconds for all major application functions (login, account viewing, money transfers).
  • KR 5: Complete two rounds of end-to-end testing with a pass rate of at least 98% to ensure platform stability.

Objective 5: Build a Good Product Knowledge Base

  • KR 1: Create a complete user manual covering all MVP features and functionalities by the end of the project.
  • KR 2: Document API endpoints.
